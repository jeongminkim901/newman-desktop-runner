      사용 설명서 (단계별)
      
          
      Postman 컬렉션을 로컬에서 실행하고 리포트/로그를 저장합니다. 탐색 모드는 변형 요청으로 품질을 점검합니다.
      

          
      1단계: 필수 입력
      
          
        
            
      1. Collection *에서 Postman 컬렉션 JSON을 선택합니다.
        2. 
            
      2. Output Directory *에 리포트/로그 저장 폴더를 지정합니다.
        3. 
          
      

          
      2단계: 선택 입력
      
          
        
            
      1. Environment (optional)을 넣으면 변수 값이 자동 적용됩니다.
        2. 
            
      2. IP와 Token을 입력하면 요청에 치환/헤더가 자동 적용됩니다.
        3. 
            
      3. Extra Vars로 추가 변수 JSON을 넣을 수 있습니다.
        4. 
          
      

          
      3단계: 실행 모드 선택
      
          
        
            
      1. Run Newman만 켜면 일반 실행입니다.
        2. 
            
      2. Exploratory API test를 켜면 변형 요청 테스트가 수행됩니다.
        3. 
          
      

          
      4단계: 탐색 옵션 설정
      
          
        
            
      1. Variants per request: 요청당 변형 개수(기본 3, 최대 5)
        2. 
            
      2. Delay (ms): 요청 사이 지연(기본 300ms)
        3. 
            
      3. Rule mode:
              Basic(필드삭제/빈값/타입깨기), Extended(경계/특수문자 포함), Custom(직접 JSON)
            
        4. 
            
      4. Variant template 선택 시 Custom JSON이 자동 채워집니다.
        5. 
            
      5. Re-explore failed only: 마지막 JSON Preview의 실패 요청만 재탐색
        6. 
          
      

          
      5단계: 컬렉션 요청 선택
      
          
        
            
      1. 트리에서 실행할 요청을 선택하거나 Search로 필터링합니다.
        2. 
            
      2. Run selected requests only를 켜면 선택된 요청만 실행됩니다.
        3. 
          
      

          
      6단계: 실행
      
          
        
            
      1. Run Newman 버튼을 누릅니다.
        2. 
            
      2. 상태줄과 Live Logs에 진행 상황이 표시됩니다.
        3. 
          
      

          
      7단계: 결과 확인
      
          
        
            
      1. History에서 실행 결과를 확인합니다.
        2. 
            
      2. Open/Preview 버튼으로 HTML/JSON/Log를 볼 수 있습니다.
        3. 
            
      3. Explore 런은 EXPLORE 배지로 표시됩니다.
        4. 
          
      

          
      8단계: 실패만 재탐색
      
          
        
            
      1. 먼저 JSON Preview를 한 번 엽니다.
        2. 
            
      2. Re-explore failed only 체크 후 다시 실행합니다.
        3. 
            
      3. 이때 Rule mode는 자동으로 Extended로 동작합니다.
        4. 
          
      

          
      자주 묻는 문제
      
          
        
            
      • HTML이 비면: HTML Reporter 체크 확인
        • 
            
      • 파일 열기 실패: Output Directory 권한 확인
        • 
            
      • 실패만 재실행이 안됨: JSON Preview를 먼저 열어야 함
        •
