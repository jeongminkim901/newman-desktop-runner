      사용 설명서 (단계별)
      
              
            
      
            
      
              
      Postman 컬렉션을 로컬에서 실행하고 리포트/로그를 저장합니다. 탐색 모드는 변형 요청으로 품질을 점검합니다.
      

              
      1단계: 필수 입력
      
              
        
                
      1. 컬렉션 *에서 Postman 컬렉션 JSON을 선택합니다(또는 OpenAPI 사용).
        2. 
                
      2. 출력 폴더 *에 리포트/로그 저장 폴더를 지정합니다.
        3. 
              
      

              
      2단계: 선택 입력
      
              
        
                
      1. 환경(선택)을 넣으면 변수 값이 자동 적용됩니다.
        2. 
                
      2. IP와 토큰을 입력하면 요청에 치환/헤더가 자동 적용됩니다.
        3. 
                
      3. 추가 변수로 추가 변수 JSON을 넣을 수 있습니다.
        4. 
                
      4. OpenAPI/Swagger 파일 또는 URL로 컬렉션을 생성할 수 있습니다.
        5. 
                
      5. 서버 선택/직접 입력으로 기본 URL을 지정할 수 있습니다.
        6. 
              
      

              
      3단계: 실행 모드 선택
      
              
        
                
      1. 실행 버튼을 누르면 일반 실행입니다.
        2. 
                
      2. 탐색적 API 테스트를 켜면 변형 요청 테스트가 수행됩니다.
        3. 
              
      

              
      4단계: 탐색 옵션 설정
      
              
        
                
      1. 요청당 변형 수: 요청당 변형 개수(기본 3, 최대 5)
        2. 
                
      2. 지연 (ms): 요청 사이 지연(기본 300ms)
        3. 
                
      3. 규칙 모드:
                  기본(필드삭제/빈값/타입깨기), 확장(경계/특수문자 포함), 사용자(직접 JSON)
                
        4. 
                
      4. 변형 템플릿 선택 시 사용자 JSON이 자동 채워집니다.
        5. 
                
      5. 실패만 재탐색: 마지막 JSON 미리보기의 실패 요청만 재탐색
        6. 
                
      6. 메서드 변형 포함: HTTP 메서드 변형 실행 여부
        7. 
                
      7. SSL 검증 무시: 사설 인증서/내부망 HTTPS 오류가 날 때 사용(보안상 운영환경 비추천)
        8. 
              
      

              
      5단계: 컬렉션 요청 선택
      
              
        
                
      1. 트리에서 실행할 요청을 선택하거나 검색으로 필터링합니다.
        2. 
                
      2. 선택된 요청만 실행을 켜면 선택된 요청만 실행됩니다.
        3. 
              
      

              
      6단계: 실행
      
              
        
                
      1. 실행 버튼을 누릅니다.
        2. 
                
      2. 상태줄과 실시간 로그에 진행 상황이 표시됩니다.
        3. 
              
      

              
      7단계: 결과 확인
      
              
        
                
      1. 실행 이력에서 결과를 확인합니다.
        2. 
                
      2. 열기/미리보기 버튼으로 HTML/JSON/Log를 볼 수 있습니다.
        3. 
                
      3. 탐색 실행은 EXPLORE 배지로 표시됩니다.
        4. 
              
      

              
      8단계: 실패만 재탐색
      
              
        
                
      1. 먼저 JSON 미리보기를 한 번 엽니다.
        2. 
                
      2. 실패만 재탐색 체크 후 다시 실행합니다.
        3. 
                
      3. 이때 규칙 모드는 자동으로 확장으로 동작합니다.
        4. 
              
      

              
      자주 묻는 문제
      
              
        
                
      • HTML이 비면: HTML 리포터 체크 확인
        • 
                
      • 파일 열기 실패: 출력 폴더 권한 확인
        • 
                
      • 실패만 재실행이 안됨: JSON 미리보기를 먼저 열어야 함
        •
